Tabelas Hash, Ordenação Externa e Árvores

1. Introdução à Pesquisa

Antes de Hashing, é fundamental entender algoritmos de pesquisa básicos que operam em vetores.

  • Busca Sequencial: O método mais simples. Percorre o vetor do início ao fim, comparando cada elemento com a chave de busca. Complexidade: O(n).
  • Busca Binária: Um método muito mais eficiente, mas que exige que o vetor esteja ordenado. Compara a chave com o elemento do meio. Se for igual, encontrou. Se for menor, busca na metade esquerda. Se for maior, busca na metade direita. Complexidade: O(log n).

2. Hashing ou Tabelas Hash

É uma estrutura que otimiza a busca para uma complexidade média de O(1). A ideia é usar uma Função de Hashing para calcular um índice (endereço) no vetor a partir da própria chave de busca, permitindo acesso quase direto ao dado.

3. Tratando Colisões

Uma colisão ocorre quando duas chaves diferentes geram o mesmo índice. Existem duas grandes estratégias para lidar com isso:

  • Encadeamento Separado: Cada posição do vetor aponta para uma lista ligada. Todos os elementos que colidem no mesmo índice são armazenados nessa lista. É a abordagem mais flexível.
  • Endereçamento Aberto: Tenta encontrar uma posição alternativa livre no próprio vetor. Inclui métodos como Sondagem Linear (procura a próxima casa livre), Quadrática e Hashing Duplo.

1. Memória Secundária

Refere-se a dispositivos de armazenamento não volátil, como HDs e SSDs. O acesso a esses dispositivos é milhares de vezes mais lento que o acesso à memória RAM. Por isso, algoritmos que operam em dados muito grandes precisam ser projetados para minimizar o número de leituras e escritas em disco.

2. Ordenação em Memória Secundária

O objetivo é ordenar arquivos que não cabem inteiramente na RAM. O principal algoritmo para isso é o MergeSort Externo.

  1. Fase 1: Criação de "Runs" (Blocos Ordenados): O arquivo grande é lido em pedaços que caibam na RAM. Cada pedaço é ordenado internamente (com QuickSort, por exemplo) e salvo como um arquivo temporário ordenado no disco.
  2. Fase 2: Intercalação (k-way merge): Os "runs" são combinados. O algoritmo lê o início de cada run, encontra o menor elemento entre eles, escreve-o no arquivo de saída final e avança no run correspondente. O processo se repete até que todos os runs sejam intercalados.

1. Pesquisa em Memória Secundária

Para buscar dados em arquivos gigantes de forma eficiente, usamos técnicas de indexação. O Acesso Sequencial Indexado cria um índice menor, que pode ser carregado na memória, para apontar para a localização aproximada dos dados no arquivo principal, reduzindo drasticamente o tempo de busca.

2. Árvores de Pesquisa: Árvore B

As Árvores B (e suas variantes como B+) são a estrutura de dados padrão para indexação em bancos de dados. Elas são "baixas e largas", significando que cada nó armazena muitas chaves. Isso minimiza a altura da árvore e, consequentemente, o número de acessos a disco necessários para encontrar um dado.

3. Árvores Digitais

São árvores otimizadas para armazenar e buscar strings (palavras).

  • Trie (Árvore de Prefixo): Cada nó representa um caractere, e o caminho da raiz até um nó forma um prefixo. Ótima para aplicações de autocomplete.
  • Árvore Patricia: Uma versão compactada da Trie, onde nós com apenas um filho são fundidos para economizar espaço.
